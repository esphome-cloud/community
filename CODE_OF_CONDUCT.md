# Code of Conduct

## Base policy

This project adopts the **[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)**
as the baseline. The canonical text is the source of truth for the universal portion;
the additions below are project-specific layers on top of that baseline.

Read the full Contributor Covenant 2.1 at the link above before participating.

## Reporting

Send reports to **conduct@esphome.cloud**. The mailbox is monitored by the founder. Reports
are confidential by default. Public disclosure of a report or its outcome happens only with
the reporter's consent.

The response SLA is the same as the security mailbox: acknowledgement within **24 hours**,
follow-up within the week.

## Project-specific additions

These 5 clauses extend the Contributor Covenant for this repository. They are not
overrides; they are additional norms reflecting how this repo is operated (single
maintainer, AI-assisted triage, async-only).

### 1. Be patient

This repo is operated by one founder, with AI handling the first reply on most threads.
The human follow-up happens during **Tuesday office hours (14:00-16:00 UTC+8)**.
"Bumping" a thread before that window does not move it up the queue and pulls attention
away from other contributors. Patience is a contribution; impatience is friction.

### 2. Be useful

When you file an Issue or open a Discussion, lead with the smallest reproduction or the
clearest statement of intent you can manage. A 3-line bug report with a Job ID is more
useful than a 30-line bug report without one. A feature request that names the user
problem is more useful than one that names a feature.

### 3. Don't @ the maintainer

GitHub `@`-mentions notify the maintainer the same way regardless of the day or hour.
Outside Tuesday office hours, `@`-mentions to the maintainer (or to the
`esphome-cloud/maintainers` team) are treated as out-of-band pings and may be removed.
The two exceptions: (a) a security report that arrived in a public thread by mistake
(re-file to security@esphome.cloud), and (b) coordination on an already-assigned issue
the maintainer asked you to ping them on.

### 4. English or Chinese

Both English and Chinese (中文) are first-class. File issues, write discussions, and
comment in whichever you are most fluent in. Do not require other contributors to
translate. AI triage and the maintainer both read both languages. Other languages are
welcome too; if AI triage cannot route a non-EN/non-ZH thread, it will ask politely
for an English or Chinese summary.

### 5. Harassment zero-tolerance

Harassment, doxxing, targeted insults, sustained disruption, or any conduct that the
Contributor Covenant 2.1 classifies as unacceptable lead to immediate removal — first
offense, no warnings, no negotiation. This applies equally across Issues, Discussions,
review comments, and email threads under this project's umbrella.

## Enforcement scope

This Code applies to all project spaces: this GitHub repository, the Gitee mirror,
the four project mailboxes (`feedback@`, `security@`, `hello@`, `conduct@`), and any
public communication where someone is representing the project (talks, demos,
collaborations on user repos that link back here).

It does NOT extend to private channels between contributors that don't reference
the project.

## Amendments

Material changes to this document open an Issue tagged `governance` and require 14 days
of public comment before merge. The Contributor Covenant 2.1 baseline is not negotiable;
the 5 project-specific clauses above are.
